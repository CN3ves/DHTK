User Guide¶

Start DHTK¶

Loading DHTK¶

As a Python module, DHTK can be accessed from any Python console or environment by importing it as normally.

import dhtk
# ###################################################
# Welcome to the Digital Humanities ToolKit,
# the  user - friendly Python API for
# Digital Humanities research.
#
# For more information, please visit DHTK 's website:
# [https://dhtk.unil.ch/]
# ####################################################

Setting up Configurations¶

The first step to use DHTK is to set the configurations to use by calling dhtk.start()

configs = dhtk.start()

This function instantiates DHTK’s settings manager class, which searches for the user’s saved settings. It automatically prints all current settings for the user’s information.

# Current DHTK settings:
# [user]: me
# [wd]: /Users/me/DHTK/me
# [endpoint]: localhost
# [dataset]:
# [modules]: {}
# [_file]: /Users/me/DHTK/me/me.config
# [logger]: <Logger dhtk (DEBUG)>
# [_port]: 3030
# [_connect]: 30

DHTK contains a number of options to help users start up faster and automatically, most of which have a default value. The settings to pay more attention to are the following:

  • [user]: By default, DHTK tries to access the system’s information to identify the user. It is possible to change this default user name, which may be particularly relevant when working on shared machines.

  • [wd]: DHTK uses the user’ home directory to set up its working directory. This can be modified if another directory would be more suitable.

!!!

Note

DHTK uses both wd and user to search for saved settings. If different values are passed to DHTK, the user’s previous settings may not be properly acessible.

  • [endpoint]: DHTK uses a local Fuseki server as its default SPARQL endpoint for querying pre-processed datasets. To use a remote (on-line) endpoint, it can be done by adding the URL (link) to the remote endpoint.

  • [dataset]: DHTK is a modular package to which extensions can be easily added. This setting determines which of the available extension modules DHTK should load.

!!!

Note

As illustrated above, dataset does not have a default value and must be set the first time a user runs DHTK.

See also

Start function

Modifying configurations¶

Configurations can be set directly by passing their values as keyword arguments to the dhtk.start() function.

configs = dhtk.start(user="Picca",
                     wd="MyDir",
                     endpoint="http://dhtk.unil.ch/gutenberg/sparql",
                     dataset="gutenberg")
# Current DHTK settings:
# [user]: Picca
# [wd]: MyDir/Picca
# [endpoint]: http://dhtk.unil.ch/gutenberg/sparql
# [dataset]: gutenberg
# [modules]: {}
# [_file]: MyDir/Picca/Picca.config
# [logger]: <Logger dhtk (DEBUG)>
# [_port]: 3030
# [_connect]: 30

Alternatively, all settings can be modified using Python’s dot notation to access the configuration object’s attributes. For example, updating the user name is as simple as configs.user = “egloff”

DHTK extensions¶

Using the settings, DHTK will determine which SPARQL endpoint to connect and which extension module to load. These two functionalities are all handled by the function dhtk.get_module().

Currently, DHTK includes the following extensions:

  • Project Gutenberg

  • Auchinkleck manuscript

See also

get_module() function

Gutenberg¶

DHTK’s Gutenberg extension dataset allows the user to easily query Gutenberg.org for books, authors, bookshelves and subjects, retrieving all the available information as a DHTK Corpus.

dhtk_configurations.dataset
# 'gutenberg'

gutenberg = dhtk.get_module(dhtk_configurations)
# DHTK Gutenberg statistics:
# Last update: 26 Dec 2020
# number of books        :	 56913
# number of authors      :	 21031
# number of bookshelves  :	   337
# number of subjects     :	 34045

DHTK provides a simple dictionary-like syntax to search Guntenberg’s dataset using the method .get(). This method allows the user to easily query the entire dataset using a single interface with only 3 keyword arguments:

  • [what] to search for.

  • [name] of what is being searched. If no name is provided, all available options are retrieved.

  • [add] to corpus or just search results?

See also

Module.get()


Exploring Data¶

To search Gutenberg’s data, set add=False. For such queries, DHTK Gutenberg’s module returns a dictionary of books or authors matching the query.

!!!

Note

add=False is the default value, so this argument can be simply ignored by the user in such cases.

Searching for a bookshelf

To check which bookshelves are available in DHTK’s Gutenberg, set what=”shelf” and name=”all”.

!!!

Note

name=”all” is the default value, so this argument may be simply ignored by the user.

# Search all bookshelves
bookshelves = gutenberg.get(what="shelf")
[shelf for shelf in bookshelves if "Science" in shelf]
# ['FR Science fiction', 'FR Sciences et Techniques',
# 'Popular Science Monthly', 'Precursors of Science Fiction',
# 'Science', 'Science Fiction', 'Science Fiction by Women',
# 'The International Magazine of Literature, Art, and Science']

To search for books available for a bookshelf, the name, or part of the name, of an available bookshelf is passed to the name argument. For example, to search all books in “Science Fiction” bookshelves:

# Search bookshelf example
bookshelves = gutenberg.get(what="shelf", name="Science Fiction")

The list of bookshelves matching the query are stored as the dictionary keys for easy access:

# Matching bookshelves found
bookshelves.keys()
# dict_keys(['Science Fiction', 'Science Fiction by Women', 'FR Science fiction',
# 'Precursors of Science Fiction'])

The information for individual bookshelves can be easily retrieved using the dictionary key. This strutcture makes it easy to perform all types of regular Python operations. For example, to identify all books that are simultaneously found on the bookshelves “Precursors of Science Fiction” and “Science Fiction by Women”:

# Intersect two bookshelves
set(bookshelves["Precursors of Science Fiction"]).intersection(bookshelves["Science Fiction by Women"])
# {'The Last Man [Shelley, Mary Wollstonecraft]',
# 'Frankenstein; Or, The Modern Prometheus [Shelley, Mary Wollstonecraft]'}

Searching for a subject

Gutenberg subjects can be searched using what=”subject”. Subject names are structured hierarchically, making it possible to define the granularity of such queries. Each hierarchy is separated by -- (e.g. Subject -- Sub-subject -- Sub-sub-subject)

# Search subject example
subjects = gutenberg.get(what="subject", name="Gothic fiction")

# Matching bookshelves found
subjects.keys()
# dict_keys(['Gothic fiction', 'Gothic fiction--Parodies, imitations, etc.'])

# Check some book examples
subjects['Gothic fiction'][-15:-10]
# ['The Old English Baron: a Gothic Story [Reeve, Clara]', 'The Children of the Abbey: A Tale [Roche, Regina Maria]',
# 'Varney the Vampire; Or, the Feast of Blood [Rymer, James Malcolm]',
# 'Frankenstein; Or, The Modern Prometheus [Shelley, Mary Wollstonecraft]',
# 'Emmeline, the Orphan of the Castle [Smith, Charlotte]']

Searching for a book

Gutenberg books can be searched using the what=”book” argument. This will return a dictionary of books containing the text passed as name. To avoid long names, the dictionary keys returned are truncated at 20 characters.

# Search book example
books = gutenberg.get(what="books", name="Frankenstein")
books
# {'Frankenstein; Or, Th': <Book: Mary Wollstonecraft Shelley - Frankenstein; Or, The Modern Prometheus (http://www.gutenberg.org/ebooks/84)>}

Searching for an author

Gutenberg books can be searched using the what=”author” argument.

# Search author example
authors = gutenberg.get(what="author", name="Shelley")
authors
# {'Percy Bysshe Shelley': <Author: Shelley, Percy (http://www.gutenberg.org/2009/agents/1529>),
# 'Henry Charles Shelley': <Author: Shelley, Henry (http://www.gutenberg.org/2009/agents/2162>),
# 'Mary Wollstonecraft Shelley': <Author: Shelley, Mary (http://www.gutenberg.org/2009/agents/61>)}

Check metadata¶

DHTK Guntenberg’s module uses books and authors objects to store all the relevant information retrieved. These objects can be accessed directly from the dictionaries obtained while searching as described above.

!!!

Note

The book and author objects are not returned when searching for bookshelves and subjects. These queries are considered more exploratory and instantiating all objects would be less efficient in respect to both performance and memory.

Get author information

Author objects are retrieved with author searches as described above. However, book objects also contain the author information as attribute. Because of this, there are two ways to access an author object:

# Explore author metadata
book = books['Frankenstein; Or, Th'].get_author() # author from GutenbergBook object
author = authors['Mary Wollstonecraft Shelley'] # author from GutenbergAuthor object

author == book
# True

Accessing the author’s information can be done as easily as for books, using the .print_info() method:

author.print_info()

Get book text

The original text for a Gutenberg book can also be accessed if required, using the GutenbergBook method .original_text().

# Explore book's text
book.original_text()[:1000]

print(book.original_text()[1749:1961])
# I am already far north of London, and as I walk in the streets of
# Petersburgh, I feel a cold northern breeze play upon my cheeks, which
# braces my nerves and fills me with delight.  Do you understand this
# feeling?

Saving results¶

To add the retrieved book to a DHTK Corpus use add=True. A list of GutenbergBook objects is created containing all the textual data and metadata retrieved from Guntenberg.org.

!!!

Note

Each query that uses add=True will add the resulting books to the same module object. This means that a new module needs to be instantiated with .get_module() function for each different corpus

Make a Corpus

# Save data to Corpus
gutenberg.get(what="author", name="Shelley, Mary Wollstonecraft", add=True)
# {'Mary Wollstonecraft Shelley': <Author: Shelley, Mary (http://www.gutenberg.org/2009/agents/61>)}

# Check Corpus
gutenberg.corpus()
# Corpus name:
# 	Gutenberg Corpus
# Corpus description:
# 	Gutenberg Books
# Corpus path:
# 	/Users/me/DHTK/me/requests/Gutenberg_Corpus
# Corpus books:
# 0	Mary Wollstonecraft Shelley    	Frankenstein; Or, The Modern Prometheus
# 1	Mary Wollstonecraft Shelley    	History of a Six Weeks' Tour Through a Part of France, Switzerland, Germany, and Holland: With Letters Descriptive of a Sail Round the Lake of Geneva, and of the Glaciers of Chamouni.
# 2	Mary Wollstonecraft Shelley    	Mathilda
# 3	Mary Wollstonecraft Shelley    	Notes to the Complete Poetical Works of Percy Bysshe Shelley
# 4	Mary Wollstonecraft Shelley    	Proserpine and Midas
# 5	Mary Wollstonecraft Shelley    	Tales and Stories Now First Collected
# 6	Mary Wollstonecraft Shelley    	The Last Man

Change Corpus’ description

Before saving the Corpus to disk, it is important to review the Corpus descriptions, particularly the corpus name, which is used as the directory where the books will be stored. This can be done simply by passing the name and description arguments to the .corpus() method used above.

# Change Corpus settings
gutenberg.corpus(name="Shelley",
                 description="Mary Shelley's Bibliography")
# Corpus name:
# 	Shelley
# Corpus description:
# 	Mary Shelley's Bibliography
# Corpus path:
# 	/Users/me/DHTK/me/requests/Shelley
# Corpus books:
# 0	Mary Wollstonecraft Shelley    	Frankenstein; Or, The Modern Prometheus
# 1	Mary Wollstonecraft Shelley    	History of a Six Weeks' Tour Through a Part of France, Switzerland, Germany, and Holland: With Letters Descriptive of a Sail Round the Lake of Geneva, and of the Glaciers of Chamouni.
# 2	Mary Wollstonecraft Shelley    	Mathilda
# 3	Mary Wollstonecraft Shelley    	Notes to the Complete Poetical Works of Percy Bysshe Shelley
# 4	Mary Wollstonecraft Shelley    	Proserpine and Midas
# 5	Mary Wollstonecraft Shelley    	Tales and Stories Now First Collected
# 6	Mary Wollstonecraft Shelley    	The Last Man

Save Corpus

When all the information as been added to a corpus, it can be easily saved to disk using the .save() method.

# Save Corpus to disk
gutenberg.save()
